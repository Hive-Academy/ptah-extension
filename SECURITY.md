# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Ptah, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@ptah.live**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Fix & Disclosure**: Coordinated with reporter

### Scope

This policy applies to:

- The Ptah VS Code extension
- The Ptah desktop application (Electron)
- The Ptah license server (api.ptah.live)
- The Ptah website (ptah.live)

### Out of Scope

- Third-party dependencies (report to the respective maintainer)
- Social engineering attacks
- Denial of service attacks

## Supported Versions

| Version  | Supported |
| -------- | --------- |
| Latest   | Yes       |
| < Latest | No        |
