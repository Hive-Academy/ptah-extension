# TASK_2025_099: Real-Time Subagent Text Streaming via SDK Hooks

## Status: IN PROGRESS - Testing Phase

## Overview

Enable real-time text streaming from subagents (spawned by Claude) to the UI. The backend watches agent JSONL files and extracts text blocks to stream to the frontend.

---

## Architecture

```
Backend Flow:
SubagentStart Hook → AgentSessionWatcherService → startWatching()
                                                    ↓
                                            Find agent-{id}.jsonl file
                                                    ↓
                                            startTailingFile() - poll for new content
                                                    ↓
                                            extractSummaryText() - get text blocks
                                                    ↓
                                            emit('summary-chunk') with toolUseId
                                                    ↓
                                    RpcMethodRegistrationService forwards to webview
                                                    ↓
Frontend Flow:
VSCodeService receives AGENT_SUMMARY_CHUNK
                ↓
ChatStore.handleAgentSummaryChunk()
                ↓
sessionManager.getAgent(toolUseId) → find agent node
                ↓
Update node.summaryContent
                ↓
replaceNodeInTree() → update messages signal
                ↓
InlineAgentBubbleComponent re-renders with summaryContent
```

---

## Issues Encountered & Fixes Applied

### Issue 1: Agent File Matching Failure

**Symptom**: `[WARN] Could not extract sessionId from file` - file detected but not matched

**Root Cause**: The matching logic tried to read sessionId from file first line, but file was empty when first created (race condition).

**Fix Applied** (`agent-session-watcher.service.ts` lines 375-394):

```typescript
// TASK_2025_099 FIX: First try direct agentId matching (most reliable)
// The filename contains the agentId, so we can match directly
const filenameAgentId = filename.replace('agent-', '').replace('.jsonl', '');
const directMatch = this.activeWatches.get(filenameAgentId);
if (directMatch && !directMatch.agentFilePath) {
  // Direct match by agentId - no need to read file content
  directMatch.agentFilePath = filePath;
  this.startTailingFile(filenameAgentId, filePath);
  return;
}
```

### Issue 2: UI Not Rendering summaryContent

**Symptom**: Backend logs showed data reaching ChatStore, but UI only showed tool calls, not text.

**Root Cause 1**: `InlineAgentBubbleComponent` template never rendered `node().summaryContent` - it only rendered `node().children` (tool calls).

**Fix Applied** (`inline-agent-bubble.component.ts` template):

```typescript
<!-- TASK_2025_099: Show summaryContent from real-time file watcher -->
@if (hasSummaryContent()) {
  <div class="text-[11px] text-base-content/80 whitespace-pre-wrap py-2">
    {{ summaryContent() }}
    @if (isStreaming()) {
      <ptah-typing-cursor colorClass="text-base-content/60" />
    }
  </div>
}
```

**Root Cause 2**: `hasChildren()` and `hasSummaryContent()` were regular methods, not computed signals. With OnPush change detection, they didn't trigger re-renders.

**Fix Applied** (`inline-agent-bubble.component.ts` lines 317-336):

```typescript
// Before (doesn't trigger re-render)
protected hasChildren(): boolean {
  return (this.node().children?.length ?? 0) > 0;
}

// After (triggers re-render when node() changes)
readonly hasChildren = computed(() => {
  return (this.node().children?.length ?? 0) > 0;
});

readonly hasSummaryContent = computed(() => {
  const content = this.node().summaryContent;
  return !!content && content.length > 0;
});

readonly summaryContent = computed(() => {
  return this.node().summaryContent || '';
});
```

### Issue 3: User Abort Logged as ERROR

**Symptom**: When user stopped a session, `[ERROR] Claude Code process aborted by user` appeared.

**Fix Applied** (`stream-transformer.ts` and `chat-rpc.handlers.ts`):

- Detect abort patterns: `"aborted by user"`, `"abort"`, `"cancelled"`, `"canceled"`
- Log at INFO level instead of ERROR for user-initiated aborts

## Key Insights

1. **Direct agentId matching is reliable** - Filename `agent-{agentId}.jsonl` matches the agentId from SubagentStart hook
2. **OnPush + signals require computed()** - Regular methods don't trigger re-renders
3. **Tree immutability matters** - Must create new node references for signals to detect changes
4. **User aborts should be INFO not ERROR** - Expected behavior, not an error
